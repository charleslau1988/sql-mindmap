▶ 一、SQL 五大分类（必背）DDL/DML/DCL/DQL/TCL
▶ 分类总览
缩写全称中文作用关键字
DDLData Definition Language数据定义语言定义数据库对象结构CREATE, ALTER, DROP, TRUNCATE
DMLData Manipulation Language数据操作语言增删改表中数据INSERT, UPDATE, DELETE
DCLData Control Language数据控制语言管理用户权限GRANT, REVOKE
DQLData Query Language数据查询语言查询数据SELECT
TCLTransaction Control Language事务控制语言控制事务提交回滚COMMIT, ROLLBACK, SAVEPOINT
▶ 二、基础查询SELECT/WHERE/ORDER BY/LIMIT
▶ SELECT（查询） 
SELECT 列 FROM 表 WHERE 条件 GROUP BY 列 HAVING 条件 ORDER BY 列;
▶ WHERE（条件筛选）
=、!=、<、>、<=、>=BETWEEN...AND（范围）IN(...)（枚举）LIKE（模糊匹配 % _）IS NULL / IS NOT NULLAND（且）、OR（或）、NOT（非）
▶ ORDER BY（排序） 
ORDER BY 列 ASC;        -- 升序（默认）
ORDER BY 列 DESC;       -- 降序
ORDER BY 列1 ASC, 列2 DESC;
▶ LIMIT / TOP（分页） 
-- MySQL / PostgreSQL
LIMIT 10 OFFSET 0;

-- SQL Server
TOP 10;
▶ 三、聚合函数（Aggregate Functions）5个函数
▶ COUNT / SUM / AVG / MAX / MIN
函数全称作用
COUNT()Count统计行数
SUM()Sum求和
AVG()Average求平均值
MAX()Maximum求最大值
MIN()Minimum求最小值
▶ 四、分组与过滤GROUP BY + HAVING
▶ GROUP BY（分组） 
SELECT category, COUNT(*) FROM products GROUP BY category;
▶ HAVING（分组后过滤） 
SELECT category, AVG(price)
FROM products
GROUP BY category
HAVING AVG(price) > 100;
WHERE：过滤原始数据HAVING：过滤聚合结果
▶ 五、表连接（JOIN）5种连接
▶ JOIN 类型
连接类型说明
INNER JOIN只返回两表匹配的行
LEFT JOIN返回左表全部 + 右表匹配的
RIGHT JOIN返回右表全部 + 左表匹配的
FULL JOIN返回两表全部（匹配+不匹配）
CROSS JOIN笛卡尔积（不常用）
-- 显式写法（推荐）
SELECT * FROM 表1 INNER JOIN 表2 ON 条件;

-- 隐式写法
SELECT * FROM 表1, 表2 WHERE 条件;
▶ 六、子查询（Subquery）5种类型
▶ 子查询类型
类型说明
标量子查询 (Scalar)返回单个值
列子查询 (Column)返回一列
表子查询 (Table)返回临时表
EXISTS 子查询检查是否存在
IN 子查询检查是否在列表中
-- 标量
SELECT * FROM users WHERE age > (SELECT AVG(age) FROM users);

-- IN
SELECT * FROM orders WHERE user_id IN (SELECT id FROM vip_users);

-- EXISTS
SELECT * FROM orders o WHERE EXISTS (SELECT 1 FROM users u WHERE u.id = o.user_id);
▶ 七、约束（Constraint）10种约束
▶ 约束总览
PRIMARY KEY：主键，唯一且非空FOREIGN KEY：外键，关联另一表UNIQUE：唯一约束NOT NULL：非空约束CHECK：条件检查DEFAULT：默认值INDEX：索引，加速查询AUTO_INCREMENT：自动递增（MySQL）IDENTITY：自动递增（SQL Server）SERIAL：自动递增（PostgreSQL）
▶ 八、DML（数据操作）INSERT/UPDATE/DELETE
▶ INSERT（插入） 
INSERT INTO 表 (列1, 列2) VALUES (值1, 值2);
INSERT INTO 表 SELECT ...;           -- 从查询插入
▶ UPDATE（更新） 
UPDATE 表 SET 列1 = 值1 WHERE 条件;
UPDATE 表 SET 列1 = 值1, 列2 = 值2;
▶ DELETE（删除） 
DELETE FROM 表 WHERE 条件;
DELETE FROM 表;                      -- 删除全部
TRUNCATE TABLE 表;                   -- 清空表（更快，不可回滚）
▶ 九、DDL（数据定义）CREATE/ALTER/DROP
▶ CREATE（创建） 
CREATE DATABASE 数据库名;
CREATE TABLE 表名 (
    id INT PRIMARY KEY,
    name VARCHAR(50) NOT NULL,
    age INT DEFAULT 18,
    email VARCHAR(100) UNIQUE
);
▶ ALTER（修改） 
ALTER TABLE 表名 ADD 列名 类型;
ALTER TABLE 表名 DROP COLUMN 列名;
ALTER TABLE 表名 MODIFY 列名 新类型;
ALTER TABLE 表名 RENAME TO 新表名;
▶ DROP（删除） 
DROP DATABASE 数据库名;
DROP TABLE 表名;

▶ 视图操作 
CREATE VIEW v_user AS SELECT name, age FROM users;
CREATE VIEW：创建视图DROP VIEW：删除视图WITH CHECK OPTION：限制更新范围
▶ 十一、事务（Transaction）ACID
▶ 事务控制语句
关键字说明
BEGIN开始事务
COMMIT提交事务
ROLLBACK回滚事务
SAVEPOINT设置保存点
BEGIN;
UPDATE account SET balance = balance - 100 WHERE id = 1;
UPDATE account SET balance = balance + 100 WHERE id = 2;
COMMIT;  -- 或 ROLLBACK;
▶ ACID 特性
特性说明
原子性 (Atomicity)要么全成功，要么全失败
一致性 (Consistency)事务前后数据一致
隔离性 (Isolation)并发事务互不干扰
持久性 (Durability)提交后数据永久保存
▶ 十二、索引（INDEX）5种索引
▶ 索引类型
类型说明
B-Tree默认索引，范围查询快
Hash等值查询快
Unique唯一索引
Composite复合索引，多列组合
Full-Text全文索引
CREATE INDEX idx_name ON 表名(列名);
CREATE UNIQUE INDEX idx_name ON 表名(列名);
CREATE INDEX idx_name ON 表名(列1, 列2);  -- 复合索引
DROP INDEX idx_name ON 表名;

▶ 十四、高级特性窗口函数/CTE/UNION
▶ 窗口函数（Window Function） 
SELECT name, salary,
    SUM(salary) OVER (PARTITION BY dept ORDER BY id) AS running_total
FROM employees;
函数说明
ROW_NUMBER()行号（不重复）
RANK()排名（可并列）
DENSE_RANK()紧凑排名
NTILE()分组排名
LAG() / LEAD()前后行值
SUM() OVER()窗口求和
▶ CTE（Common Table Expression）

    SELECT * FROM users WHERE age > 20
)
SELECT * FROM temp;

▶ 十五、SQL 执行顺序（背这个！）口诀
▶ 执行顺序
执行顺序：

1️⃣ FROM    -- 确定表
2️⃣ WHERE   -- 筛选行
3️⃣ GROUP BY -- 分组
4️⃣ HAVING   -- 分组后筛选
5️⃣ SELECT  -- 选择列（计算表达式）
6️⃣ ORDER BY -- 排序
7️⃣ LIMIT    -- 限制行数
口诀：FROM WHERE GROUP，HAVING SELECT ORDER LIMIT
